typedef void(* mxuvc_audio_alert_cb_t)(audio_alert_info *audalert_info, void *user_data)

Callback function to register with MXUVC for getting alert notification.

Parameters:
[out]audalert_infopointer to audio alert structure audio_alert_info which holds the details of the alert event generated by the firmware
[out]user_datapointer to user context/ user data that the user wants to access inside the callback function. The pointer value is returned as given to MXUVC at the time of registering the callback function.

int mxuvc_overlay_privacy_add_mask ( video_channel_t  ch,
privacy_params_t *  params,
privacy_index_t  idx 
)

API to add a privacy mask on a specific video channel.

This API allows to add a privacy mask on the video channel using GPU hardware. Number of indexes per channel is controlled by PRIVACY_IDX_MAX and should be less than the same. mxuvc_overlay_privacy_remove_mask must be called before re-adding mask on same ch and idx values. Error will be returned if called consecutively without removing the mask.

Parameters:
chvideo channel on which privacy mask needs to be applied.
paramsdata structure containing properties of the mask like shape, type, color.
idxindex to control a specific mask on same channel.
Returns:
0 on Success, -1 on Failure
int mxuvc_overlay_privacy_remove_mask ( video_channel_t  ch,
privacy_index_t  idx 
)

API to remove a privacy mask on a specific video channel.

This API allows to remove a specific mask from the selected video channel. mxuvc_overlay_privacy_add_mask must be called before calling this api. Error will be returned if called consecutively without adding the mask.

Parameters:
chvideo channel from which privacy mask needs to be removed.
idxindex of the mask that needs to removed.
Returns:
0 on Success, -1 on Failure
int mxuvc_overlay_privacy_update_color ( video_channel_t  ch,
privacy_color_t *  colors,
privacy_index_t  idx 
)

API to update the color and alpha of applied mask. Not Implemented Yet.

Not Implemented Yet. API allows to change the color and alpha of selected mask on specific video channel. mxuvc_overlay_privacy_add_mask must be called before calling this API else error will be returned. color is a 32 bit value with first 24 bit representing color in YUV format and last 8 bit representing alpha or transparency between 0x00-0xFF. Where 0xFF is opaque and 0x00 means transparent.

Parameters:
chvideo channel from which color needs to be updated
colorscolorkey in YUVA format.
idxindex of the mask that needs to updated.
Returns:
0 on Success, -1 on Failure
